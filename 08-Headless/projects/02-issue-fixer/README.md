# 项目 2：Issue 自动修复

> 分析 GitHub Issue 并自动创建修复 PR

---

## 场景说明

当用户创建 Bug Issue 时，自动：
1. 分析 Issue 内容
2. 定位问题代码
3. 生成修复方案
4. 创建修复 PR

## GitHub Actions 工作流

### .github/workflows/issue-to-pr.yml

```yaml
name: Issue Auto Fix

on:
  issues:
    types: [labeled]

permissions:
  contents: write
  pull-requests: write
  issues: write

jobs:
  auto-fix:
    if: github.event.label.name == 'auto-fix'
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Setup Claude Code
        run: npm install -g @anthropic-ai/claude-code

      - name: Analyze Issue
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
          ISSUE_TITLE: ${{ github.event.issue.title }}
          ISSUE_BODY: ${{ github.event.issue.body }}
          ISSUE_NUMBER: ${{ github.event.issue.number }}
        id: analyze
        run: |
          # 分析 Issue 并生成修复
          claude -p "
          A GitHub Issue needs to be fixed:

          Title: $ISSUE_TITLE
          Description: $ISSUE_BODY

          Please:
          1. Understand the problem
          2. Locate the relevant code
          3. Propose a fix
          4. Only modify files in the allowed list

          Do NOT make any changes yet, just analyze and report your plan.
          " \
            --allowedTools "Read,Grep,Glob" \
            --output-format json \
            --max-turns 10 \
            > analysis.json

          echo "analysis<<EOF" >> $GITHUB_OUTPUT
          cat analysis.json | jq -r '.result' >> $GITHUB_OUTPUT
          echo "EOF" >> $GITHUB_OUTPUT

      - name: Create Fix Branch
        run: |
          git config user.name "claude-bot"
          git config user.email "claude-bot@anthropic.com"
          git checkout -b fix/issue-${{ github.event.issue.number }}

      - name: Apply Fix
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
          ISSUE_TITLE: ${{ github.event.issue.title }}
          ISSUE_BODY: ${{ github.event.issue.body }}
        run: |
          claude -p "
          Fix this GitHub Issue:

          Title: $ISSUE_TITLE
          Description: $ISSUE_BODY

          Apply the necessary code changes to fix this issue.
          Only modify files that are safe to modify.
          " \
            --allowedTools "Read,Edit,Write,Grep,Glob" \
            --disallowedTools "Bash" \
            --max-turns 15

      - name: Create Pull Request
        run: |
          git add -A
          git commit -m "Fix: ${{ github.event.issue.title }}

          Closes #${{ github.event.issue.number }}

          This fix was automatically generated by Claude Code."

          git push origin fix/issue-${{ github.event.issue.number }}

          gh pr create \
            --title "Fix: ${{ github.event.issue.title }}" \
            --body "
          ## Summary
          This PR fixes the issue #${{ github.event.issue.number }}

          ## Analysis
          ${{ steps.analyze.outputs.analysis }}

          ---
          *This PR was automatically generated by Claude Code*
          " \
            --base main
        env:
          GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}

      - name: Comment on Issue
        uses: actions/github-script@v7
        with:
          script: |
            github.rest.issues.createComment({
              issue_number: context.issue.number,
              owner: context.repo.owner,
              repo: context.repo.repo,
              body: `## 🤖 Auto-fix PR Created

              I've analyzed this issue and created a fix: PR #${{ github.event.issue.number }}

              Please review the changes.

              ---
              *Automatically generated by Claude Code*`
            });
```

## 安全配置

### safety/allowed-files.txt

```
# 允许 Claude 修改的文件
src/**/*.js
src/**/*.ts
src/**/*.jsx
src/**/*.tsx
tests/**/*.js
tests/**/*.ts
*.md
*.json

# 禁止修改
!.env*
!credentials/*
!secrets/*
!.github/workflows/*
```

## 学习要点

1. **Issue 触发**
   - 使用 `issues: types: [labeled]` 触发
   - 只有标记 `auto-fix` 的 Issue 才处理

2. **安全边界**
   - 使用 `--disallowedTools` 禁止 Bash
   - 维护允许修改的文件列表
   - 限制最大迭代次数

3. **PR 创建**
   - 使用 `gh pr create` CLI
   - 自动关联 Issue
   - 包含分析说明
