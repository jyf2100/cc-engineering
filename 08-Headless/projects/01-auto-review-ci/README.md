# 项目 1：自动代码审查 CI

> 在 PR 提交时自动运行 Claude 代码审查

---

## 场景说明

当开发者提交 PR 时，自动触发 Claude 进行代码审查：
- 检查安全漏洞
- 检查代码质量
- 提供改进建议
- 自动发布审查评论

## GitHub Actions 工作流

### .github/workflows/claude-review.yml

```yaml
name: Claude Code Review

on:
  pull_request:
    types: [opened, synchronize]

permissions:
  contents: read
  pull-requests: write
  issues: write

jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Get changed files
        id: changed-files
        run: |
          echo "files=$(git diff --name-only ${{ github.event.pull_request.base.sha }} ${{ github.sha }} | tr '\n' ' ')" >> $GITHUB_OUTPUT

      - name: Setup Claude Code
        run: |
          npm install -g @anthropic-ai/claude-code

      - name: Run Claude Review
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        id: review
        run: |
          claude -p "Review these changed files for code quality, potential bugs, and best practices: ${{ steps.changed-files.outputs.files }}" \
            --allowedTools "Read,Grep,Glob,Bash(git diff*)" \
            --output-format json \
            --max-turns 10 \
            > review_result.json

          # 提取结果
          RESULT=$(cat review_result.json | jq -r '.result')
          echo "review_result<<EOF" >> $GITHUB_OUTPUT
          echo "$RESULT" >> $GITHUB_OUTPUT
          echo "EOF" >> $GITHUB_OUTPUT

          # 检查成本
          COST=$(cat review_result.json | jq -r '.cost_usd')
          echo "Review cost: \$$COST"

      - name: Post Review Comment
        uses: actions/github-script@v7
        with:
          script: |
            const result = `${{ steps.review.outputs.review_result }}`;

            const comment = `## 🤖 Claude Code Review

            ${result}

            ---
            *This review was automatically generated by Claude Code.*

            `;

            github.rest.issues.createComment({
              issue_number: context.issue.number,
              owner: context.repo.owner,
              repo: context.repo.repo,
              body: comment
            });
```

### .github/workflows/claude-security.yml

```yaml
name: Claude Security Scan

on:
  pull_request:
    types: [opened, synchronize]
    paths:
      - '**.js'
      - '**.ts'
      - '**.py'
      - '**.go'

permissions:
  contents: read
  pull-requests: write
  security-events: write

jobs:
  security-scan:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Setup Claude Code
        run: npm install -g @anthropic-ai/claude-code

      - name: Run Security Scan
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        id: scan
        run: |
          claude -p "Scan the changed files for security vulnerabilities:
            - SQL injection
            - XSS vulnerabilities
            - Hardcoded secrets
            - Insecure dependencies
            - Authentication issues" \
            --allowedTools "Read,Grep,Glob" \
            --output-format json \
            --max-turns 5 \
            > scan_result.json

          RESULT=$(cat scan_result.json | jq -r '.result')
          IS_ERROR=$(cat scan_result.json | jq -r '.is_error')

          echo "scan_result<<EOF" >> $GITHUB_OUTPUT
          echo "$RESULT" >> $GITHUB_OUTPUT
          echo "EOF" >> $GITHUB_OUTPUT
          echo "is_error=$IS_ERROR" >> $GITHUB_OUTPUT

      - name: Post Security Comment
        if: always()
        uses: actions/github-script@v7
        with:
          script: |
            const result = `${{ steps.scan.outputs.scan_result }}`;
            const isError = `${{ steps.scan.outputs.is_error }}` === 'true';

            const emoji = isError ? '🚨' : '✅';
            const title = isError ? 'Security Issues Found' : 'Security Scan Passed';

            const comment = `## ${emoji} Claude Security Scan

            ### ${title}

            ${result}

            ---
            *Automated security scan by Claude Code*
            `;

            github.rest.issues.createComment({
              issue_number: context.issue.number,
              owner: context.repo.owner,
              repo: context.repo.repo,
              body: comment
            });

      - name: Fail if security issues found
        if: steps.scan.outputs.is_error == 'true'
        run: exit 1
```

## Prompt 模板

### prompts/review-prompt.md

```markdown
Review the code changes in this PR for:

## Code Quality
- Code organization and structure
- Naming conventions
- Code duplication
- Complexity

## Potential Bugs
- Logic errors
- Edge cases
- Null/undefined handling
- Error handling

## Best Practices
- Language-specific best practices
- Design patterns
- Testing considerations

## Actionable Feedback
Provide specific, actionable suggestions for improvement.

Format your response as:
- **Issue**: Description of the issue
- **Location**: File and line number
- **Suggestion**: How to fix it
```

## 学习要点

1. **GitHub Actions 集成**
   - 使用 `actions/checkout@v4` 获取代码
   - 使用 `actions/github-script@v7` 发布评论

2. **Headless 模式配置**
   - `--allowedTools` 限制权限
   - `--output-format json` 便于解析
   - `--max-turns` 控制成本

3. **安全性**
   - API Key 存储在 Secrets
   - 只读工具权限
   - 限制迭代次数
